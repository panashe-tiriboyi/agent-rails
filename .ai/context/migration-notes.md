# Migration Notes

Use this primer only when the project involves old-system-to-new-system work.

## Migration Inputs

- Old system source: not mapped yet.
- Target system source: not mapped yet.
- Data model: not mapped yet.
- API or workflow contracts: not mapped yet.
- Operational constraints: not mapped yet.

## Evidence Categories

- Old observed behavior
- Target observed behavior
- Documentation claim
- Approved decision
- Assumption

## Migration Rails

- Do not read the whole old system by default.
- Map one workflow, API, data area, or operational concern at a time.
- Do not treat old behavior as desired behavior when security, privacy, reliability, or product direction may differ.
- Do not accept parity differences without explicit approval.
- Record open decisions before implementation.
