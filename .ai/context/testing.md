# Testing Primer

Record test strategy, commands, and verification expectations.

## Commands

- Unit tests: not configured yet.
- Integration tests: not configured yet.
- UI or end-to-end tests: not configured yet.
- Lint/typecheck/build: not configured yet.

## Test Rails

- Match test depth to risk.
- Cover happy paths, denial paths, boundaries, malformed input, dependency failures, and regressions.
- Do not claim tests passed unless executed in the current session.
- Separate product failures, test failures, and environment failures.

## Evidence

Record durable verification evidence with date, command, result, and limitations.
