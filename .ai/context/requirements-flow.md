# Requirements Flow

Use this primer to keep implementation work intentional.

## Rule

Any requirement that changes behavior, architecture, data, security, deployment, or user workflow needs user approval before implementation.

## Draft Requirement Shape

- Request
- Business reason
- Current evidence
- Desired outcome
- Acceptance criteria
- Out of scope
- Dependencies and open decisions
- Verification plan

## Approval

Approval identifies the decision owner, date, requested behavior, and acceptance criteria. Documentation approval is not verification.

## Verification

Work is verified only when current code, tests, runtime output, or approved decisions support the claim.
